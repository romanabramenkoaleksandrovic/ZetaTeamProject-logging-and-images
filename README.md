# Проект: Стеганографическое средство сокрытия в метаданных изобранжения
**Команда разработки: ZetaEngineering**
  
Это приложение с графическим интерфейсом, предназначенное для сокрытия текстовых сообщений внутри цифровых изображений. Стеганография отличается от криптографии тем, что сам факт наличия секретного сообщения скрывается от постороннего наблюдателя. Программа предоставляет два различных метода встраивания данных:  
  
**• LSB (Least Significant Bit)** – изменение младших битов цветовых каналов пикселей. Этот метод обладает высокой ёмкостью (до 3 бит на пиксель для 24-битных изображений), но требует использования форматов без сжатия с потерями (PNG, BMP, TIFF).  
  
**• EXIF (Exchangeable Image File Format)** – запись текста в стандартные метаданные изображения (тег ImageDescription). Этот метод не изменяет видимое изображение и работает с JPEG, PNG, TIFF, но скрытое сообщение легко обнаруживается специальным ПО для анализа метаданных.  
  
Приложение разработано на .NET 6+ с использованием Avalonia UI, что обеспечивает нативный внешний вид и производительность. Для работы с изображениями используется библиотека SixLabors.ImageSharp, не требующая установки системных кодеков.  
  
# Принцип работы методов
## LSB-стеганография  
**Принцип**
В 24-битном изображении каждый пиксель описывается тремя байтами (каналы R, G, B). Человеческий глаз не различает изменение цвета при замене младшего (наименее значимого) бита в каждом канале. Алгоритм заменяет эти биты битами секретного сообщения.  
  
**Процесс встраивания (метод HideText)**  
&emsp;1. Текст преобразуется в байты UTF-8.  
&emsp;2. Добавляется нулевой байт (\0) – маркер конца сообщения.  
&emsp;3. Байты разбиваются на отдельные биты (от младшего к старшему).  
&emsp;4. Изображение загружается в память как массив пикселей Rgba32.  
&emsp;5. Для каждого пикселя последовательно заменяются младшие биты каналов R, G, B битами сообщения.  
&emsp;6. Изменённое изображение сохраняется в формате PNG (сжатие без потерь гарантирует сохранность скрытых данных).  
  
**Процесс извлечения (ExtractText)**  
&emsp;1. Изображение загружается, снова читаются младшие биты каналов R, G, B.  
&emsp;2. Из битов собираются байты.  
&emsp;3. Как только встречается нулевой байт – процесс останавливается.  
&emsp;4. Полученные байты преобразуются в строку UTF-8.  
  
**Ёмкость**  
Максимальный размер сообщения (в байтах) ≈ (Ширина × Высота × 3) / 8. Например, для изображения 1920×1080 это около 777 600 байт (760 КБ), что достаточно для больших текстов.
  
## EXIF-стеганография
**Что такое EXIF**  
Это стандарт метаданных, используемый фотокамерами и редакторами изображений. EXIF-блок хранит информацию о дате съёмки, параметрах камеры, GPS-координатах, а также различные текстовые поля, включая ImageDescription (описание изображения). Многие программы (например, просмотрщики Windows) отображают это поле в свойствах файла, но обычный пользователь редко его проверяет.
  
**Процесс встраивания (HideMessageInExif)**
&emsp;1. Проверяется существование входного файла.  
&emsp;2. Изображение загружается через Image.Load.  
&emsp;3. Из метаданных извлекается текущий EXIF-профиль, либо создаётся новый.  
&emsp;4. В профиле устанавливается значение тега ExifTag.ImageDescription равное переданной строке.  
&emsp;5. Профиль сохраняется в метаданных изображения, а само изображение – в выходной файл (обычно JPEG или PNG).  
  
**Процесс извлечения (ExtractMessageFromExif)**  
&emsp;1. Загружается изображение.  
&emsp;2. Читается EXIF-профиль.  
&emsp;3. Извлекается значение тега ImageDescription.  
&emsp;4. Если тег отсутствует или профиля нет – возвращается пустая строка.  
  
**Ограничения EXIF**  
Не все форматы поддерживают EXIF (BMP, WebP, GIF – нет или ограниченная поддержка).  
Максимальная длина текста обычно не более 64 КБ (в зависимости от реализации).  
EXIF-данные легко удаляются при открытии и повторном сохранении изображения в некоторых редакторах.  
  
# Взаимодействие с пользователем
• Выбор изображения – двумя способами:  
• Нажатие на кнопку и выбор файла через системный диалог.  
• Перетаскивание файла из проводника прямо в окно приложения.  
• Ввод сообщения – в текстовое поле можно вставить текст любой длины (ограничивается только ёмкостью изображения для LSB).  
• Шифрование – после нажатия кнопки открывается диалог сохранения файла. Приложение автоматически добавляет уникальный суффикс к имени (ZetaTeam_XXXXXXXX.png), чтобы избежать случайной перезаписи.  
• Дешифрование – после загрузки изображения со скрытым текстом и нажатия кнопки, извлечённое сообщение появляется в том же текстовом поле.  
• Обратная связь – все действия сопровождаются текстовыми сообщениями: «Изображение успешно загружено», «Ошибка: текст слишком большой», «Сообщение извлечено» и т.д.  
  
# Технические детали и производительность
• Многопоточность – операции LSB и EXIF выполняются в фоновом потоке через Task.Run, чтобы UI оставался отзывчивым.  
• Работа с памятью – ImageSharp использует Span<byte> и низкоуровневые буферы, минимизируя аллокации. Изображение загружается полностью, поэтому для огромных файлов (100+ МП) потребуется много ОЗУ.  
• Drag & Drop – реализован через события DragOver и Drop, с проверкой формата файла и расширения.  
• Тематизация – Avalonia автоматически перерисовывает все элементы при смене RequestedThemeVariant
